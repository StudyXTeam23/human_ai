===============================================
           结果页面使用指南
===============================================

✅ 已实现功能:

1. 双栏对比布局
   - 左侧: 原始文本
   - 右侧: AI 处理后的文本

2. 顶部操作栏
   - 返回按钮
   - 复制按钮 (复制处理后的文本)
   - 下载按钮 (下载为 .txt 文件)
   - 处理时间显示

3. 底部统计信息
   - 原始长度
   - 处理后长度
   - 处理时间

===============================================

📁 新建文件:

1. web/frontend/app/result/page.tsx
   - 结果展示页面
   - 双栏布局
   - 复制/下载功能

2. web/frontend/app/page.tsx (已修改)
   - 添加数据保存逻辑
   - 添加页面跳转

===============================================

🔄 工作流程:

1. 用户在主页输入文本
2. 点击 "Humanize" 按钮
3. 处理完成后保存到 localStorage
4. 自动跳转到 /result 页面
5. 结果页显示对比
6. 用户可以复制/下载/返回

===============================================

🚀 测试步骤:

1. 确保前端运行:
   ./start-frontend.sh

2. 访问主页:
   http://localhost:3000

3. 输入测试文本 (300-5000 字符)

4. 点击 "Humanize AI Text"

5. 等待处理完成

6. 自动跳转到结果页

7. 验证:
   ✓ 左侧显示原始文本
   ✓ 右侧显示处理后文本
   ✓ 复制按钮工作正常
   ✓ 下载按钮工作正常
   ✓ 返回按钮工作正常

===============================================

🎨 页面特点:

- 响应式设计
- 暗色模式支持
- 流畅的滚动体验
- 现代化 UI
- 蓝色高亮处理结果

===============================================

📝 数据传递:

使用 localStorage:
  originalText - 原始文本
  humanizedText - 处理后文本
  processingTime - 处理时间

页面加载时自动读取并清理

===============================================

🔧 如果遇到问题:

1. 页面空白?
   - 检查 localStorage 是否有数据
   - 检查浏览器控制台错误

2. 跳转失败?
   - 检查路由配置
   - 检查 /result 页面是否存在

3. 数据显示错误?
   - 清除浏览器缓存
   - 重新处理文本
